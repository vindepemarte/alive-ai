Give your AI a nervous system.

Alive-AI adds persistent mood, memory, impulses, attachment, terminal chat, Telegram, OpenMind long-term memory, and a local dashboard to a companion-style AI runtime. Use it at your own risk.

Install and run 
npx alive-ai@latest init my-ai
cd my-ai
npx . setup
npx . doctor
npx . chat
GitHub Full WebUI Setup

Local-first internal life

The runtime keeps emotional residue, weighted memories, idle thoughts, and transparent state. The dashboard is not a landing page toy; it is the same WebUI you get locally, exported statically for GitHub Pages.

Persistent affect

Emotions decay, compound, and influence future messages instead of resetting after each prompt.

Memory with weight

Built-in local memory stays active, and OpenMind can add cloud or local semantic recall.

Terminal or Telegram

Chat from the split-pane terminal with `npx . chat`, use `npx . chat --plain` for raw mode, or connect Telegram for mobile access.

Setup Flow

`npx . setup` asks for the minimum required configuration and lets optional systems be skipped. Use `local` for Ollama, `skip` for optional keys, and OpenMind cloud or local when you want a shared long-term semantic memory.

Start local terminal chat`npx . chat` starts the same runtime with chat on the left, logs on the right, and the WebUI at `http://127.0.0.1:8080`.
Start Telegram/runtime mode`npx . start` starts the configured input channel and validates Telegram before polling. Stop foreground runs with `Ctrl+C`.
Install missing tools`npx . doctor --fix` asks `y/N` for each missing tool, then uses Homebrew, winget, apt, dnf, or pacman where available.
Keep it updated`start` and `chat` check npm for newer versions. Use `npx . update` manually or `npx . uninstall` to remove local runtime files.
Use OpenMindChoose `openmind-cloud` for `https://theopenmind.pro` or `openmind-local` for `http://127.0.0.1:3333`.
Preview the dashboard`npx . demo` is keyless. The real runtime dashboard streams local state over SSE.

Full WebUI Static Export

GitHub Pages cannot run Alive-AI's FastAPI backend. The public WebUI is a standalone static export of the actual dashboard, wired to mocked local state so you can inspect the real interface without a server.

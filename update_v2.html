Alive-AI v0.2.4

Framework upgrade summary

This release turns the v0.2 architecture into a safer, more model-agnostic runtime with stronger context, cleaner extension points, local Redis memory lifecycle support, clearer startup visibility, and state-led replies without framework word or token caps.

What Changed

1

Alive body snapshot

A clean body/state contract now collects emotion, circadian, drives, memory pressure, identity, and runtime signals into a single usable snapshot.

2

Model adapter layer

Provider wiring is more universal: Ollama, OpenRouter, ZAI, OpenAI-compatible APIs, capabilities, task roles, and fallback routing are cleaner.

3

MCP tool layer

MCP now has cataloging, permission decisions, proposal storage, audit logging, and owner approval paths with sensitive argument redaction.

4

Layered memory

Memory is split into working, episodic, semantic, emotional, autobiographical, dream, procedural, and shadow layers for better context selection.

5

Behavioral pressure

Emotional state now produces action tendencies, urgency, approach/withdraw pressure, and prompt-ready behavioral guidance.

6

Plugin registry

Core systems now advertise events, state keys, prompt sections, and permissions so the runtime can grow without hidden wiring collisions.

7

Runtime lifecycle

npx . start now color-codes foreground logs and manages local Docker Redis memory when the vector cache is enabled.

8

First-chat identity

Telegram display-name metadata is stored separately from preferred identity, and direct "do you know me?" turns now get an honest memory-grounded answer.

9

State-led replies

The response shaper no longer clips visible chat by word count, sentence count, question count, or framework token budget. State guides style; the model finishes the thought.

Runtime Flow

The new architecture keeps the model from receiving random loose state. Context is compiled from structured layers before it reaches the provider.

User message Telegram, terminal, or WebUI input enters one runtime path.
Body snapshot Emotion, sleep, drives, attachment, and identity become structured context.
Memory layers Relevant facts, episodes, emotional traces, and narrative are selected.
Behavior pressure State becomes action tendency instead of only mood labels.
Model adapter The selected provider receives the right task model and capability contract.
Fixed

Local model drift

Existing Ollama installs keep their chosen model during update. Task-specific defaults no longer silently override legacy local settings.

Fixed

Unsafe MCP approvals

Denied or dangerous proposals cannot be approved into execution, and stored MCP arguments are redacted before hitting disk.

Watch

Real-world behavior

The architecture is stronger, but the real proof is long conversation testing with local Gemma and OpenRouter under the same benchmark path.

Fixed

Redis lifecycle

When Redis vector memory is enabled locally, start brings the Docker service up and stop shuts it down without deleting the persistent volume.

Fixed

Direct identity recall

Fresh installs no longer dodge "do you know me?" turns. They answer from preferred name, Telegram display name, or honestly admit there is not enough memory yet.

Fixed

Cut-feeling replies

Sleep, conflict, and low energy now guide wording without hard visible output caps. Cleanup still blocks reasoning leaks, prompt fragments, and broken tails.

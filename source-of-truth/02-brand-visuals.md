# Brand And Visuals

## Brand Identity

- Canonical product name: `Alive-AI`.
- Canonical package spelling: `alive-ai`.
- The configured agent identity is separate from the framework name. Setup seeds example identity data such as `Alice`, but the agent should speak as the configured person, not as the product.
- Origin wording in example directives: generated by Alexandru Iacovici, known as Vindepemarte, inside Alive-AI.

Status: decided from README and config examples.

## Assets

- Primary logo assets are currently under `docs/assets/`, `pages/public/assets/`, and `webui/static/`.
- Public/site favicon and app icons include `alive-ai.png`, `alive-ai-512.png`, `favicon.png`, and `apple-touch-icon.png`.
- Local WebUI avatar selection prefers local `mypics/public` portrait-like images, then falls back to `webui/static/alive-ai-512.png`.

Status: decided from file layout and `webui/app.py`.

## Existing Visual Direction

The current public Pages app and static dashboard use a dark technical/cyber companion style with high-contrast panels and diagnostic language. This direction is acceptable for now.

Future UI should keep the existing visual direction unless a deliberate redesign is requested, but colors should match the actual logo design and palette rather than drifting into unrelated blues/purples.

Status: decided by operator on 2026-06-10.

## Logo-Derived Palette

Use the actual `alive-ai-512.png` / `alive-ai.png` logo as the palette source. The logo is a chrome anatomical heart with black/transparent negative space, silver metal, blue-teal body tones, cyan highlights, and electric indigo/violet accents.

Practical palette anchors sampled from `docs/assets/alive-ai-512.png`:

- Black/near-black negative space: `#000000` where a dark background is needed.
- Chrome silver: `#D4D4DA`, `#9BA0AC`.
- Steel blue gray: `#5B6B9A`, `#57616E`.
- Deep blue/teal body tones: `#296590`, `#23576E`, `#306E8E`, `#197586`.
- Cyan/teal highlights: `#4395B5`, `#3795A4`, `#64E0F4`, `#91D9F2`.
- Electric blue/indigo/violet accents: `#4F53AB`, `#615BD6`, `#8775C6`, `#8B74D7`.

Use these colors as anchors, not as a requirement to make every screen saturated. Chrome/silver and dark negative space should carry the base; cyan/teal/violet should be accents for state, focus, glow, and active elements.

Status: decided from logo sampling and operator guidance on 2026-06-10.

## UI Rules For Future Work

- Preserve the difference between the public static portal and the local live WebUI.
- Do not replace the local WebUI with a hosted backend assumption; the live dashboard is served by the running Python runtime.
- Do not introduce a second brand name for the runtime.
- Do not treat example agent names as the product name.
- Keep the current dark technical/cyber companion style for now.
- Match UI colors to the actual logo palette: chrome/silver, dark negative space, blue-teal body tones, cyan highlights, and electric indigo/violet accents.
- If changing public Pages visuals, inspect `pages/src/` and generated `docs/` output together.
- If changing live WebUI visuals, remember `webui/static/index.html` is intentionally shipped as a static package asset.

Status: decided.
